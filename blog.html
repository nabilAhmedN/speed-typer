Basic Question about JavaScript

* LocalStorage - persists the storage across browser sessions. So even if the domain is opened in a new tab, or the browser is closed and reopened, the storage is saved. There is no time limit.
* SessionStorage - is cleared when the page session ends. A page session lasts as long as the tab or the browser is open. Therefore, when you open a new tab or a new window, this creates a new session and new storage.

* Global scope - Global scope contains all of the things defined outside of all code blocks. A code block simply consists of grouped statements inside curly braces ({ }). if statement, loops, function are examples of structure that create a code block. A global variable has global scope. A global variable is accessible from anywhere in the code.
* Local Scope - Local scope contains things defined inside code blocks. A local variable has local scope. A local variable is only accessible where itâ€™s declared.

Event Loop - to see the Callback Queue and the Call Stack. The Event Loop will pull the first event from the queue and put it to the Call Stack if it is empty, where it is effectively run. A tick is what the Event Loop refers to as such an iteration. When the function stack is empty, an event loop pulls items out of the queue and adds them to the execution stack for the function. The event loop is the trick behind JavaScript's appearance of multithreading while actually just being single-threaded.

* Uninitialized variables will return undefined.
* If there is no return, the result is undefined.
* Undefined results from a parameter that is not passed.
* If there is nothing on the right side, the return value will be undefinable.
* Undefined property is that which does not exit in an object.
* Assessing array elements outside of the array index give undefined.
